Система управления доступом к файлам на основе ролей

Архитектура:
1. Информация о ролях и правах:
   - Хранится в классе User (статический словарь RolePermissions)
   - Также можно использовать файл roles.json для конфигурации

2. Объекты доступа (файлы):
   - finance.txt - финансовая информация
   - hr.txt - кадровые данные
   - marketing.txt - маркетинговая информация

3. Регистрация пользователей:
   - Хранятся в users.json (пароли в хэшированном виде)
   - Регистрация через метод RegisterUser()

Демонстрация:
1. Запустите RoleBasedFileAccess.exe
2. Система автоматически:
   - Зарегистрирует 3 тестовых пользователя
   - Создаст текстовые файлы
   - Продемонстрирует доступ для разных ролей

Тестовые пользователи:
- Логин: admin, Пароль: admin123, Роль: Administrator
- Логин: user1, Пароль: user123, Роль: User
- Логин: user2, Пароль: user456, Роль: Guest
